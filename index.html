Duetdroid

Re-targetable Concolic Execution of Android Apps

Overview

DuetDroid is a concolic execution framework for analyzing Android apps which is designed to bridge the gap between existing Java concolic execution technique and the rapidly evolving Android Runtime (ART). DuetDroid take the position that exploit Java analysis tool to analyze Android program without any modeling of the version-specific behavior of Android Runtime. Specifcally, our framework executes Android programs based on a concolic execution tool for Java, with simultaneous concrete execution in Android.
our framework neither requires effort of mocking or modeling Android framework nor sacrificing the accuracy of app behavior due to non-real execution environment. By performing concrete execution on physical Android devices, our framework supports analysis of Android apps built using different versions of Android APIs. Empirical Study

Currently, most of Android testing techniques focus on generation of motion inputs (i.e., clicks, taps and touches) and resort to using either pre-defined or random values for data inputs (i.e., inputs such as text-boxes, edit-boxes and combo-boxes). We perform a empirical study on the AndroTest benchmark (68 Android apps). Our investigation reveals that 68% apps have data inputs.

Meanwhile, the following figure shows that percentage of data inputs in all controllable widgets. We exclude apps without any data input. On average, the input widgets occupy 25.2% of all controllable widgets.

experiment 2

We evaluate DuetDroid by assessing its ability to reveal errors in third-party libraries. Existing techniques are less effective in finding bugs in third-party libraries because (1) these libraries mainly focus on providing interfaces for the customization of Android apps but these interfaces are not tested by the current techniques and (2) testing the API provided in these libraries requires runtime modifications of the program variables which corresponds to the attributes defined in the XML. experiment 3

In this experiment, our goal is to evaluate the effectiveness of DuetDroid in finding bugs that occur due to Android fragmentation. Specifically, we evaluate the ability of DuetDroid in finding SDK-version related bugs by running the same demo app in different Android versions. The table above also shows the bugs we can find in different version. In summary, DuetDroid is able to find 6 bugs in 10 subjects, and 2 of them are version-related.
